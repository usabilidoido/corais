CCK supported modules.

If a module is not supported, the corresponding CCK fields will not be imported!

  • Amazon field (CCK Contrib)
  • Date (CCK Contrib)
  • Email (CCK Contrib)
  • File Field (CCK Contrib)
  • Image Field (CCK Contrib)
  • Number (CCK Core)
  • Phone (CCK Contrib)
  • Text (CCK Core)
  • User reference (CCK Core)

Amazon field (CCK Contrib)

The Amazon field module from the Amazon module package is fully supported.

Content fields

The Amazon field module adss the Amazon item content fields for node content types.

Content fieldFormat
Label of Amazon item fieldAny valid ASIN

Date (CCK Contrib)

The Date module is partially supported. The support currently only works for single-valued date fields, but not for multiple values nor for repeat dates.

Content fields

The Date module adds the Datetime content fields for node content types.

Content fieldFormat
Label of Datetime fieldDate

Email (CCK Contrib)

The Email module is fully supported.

Content fields

The Email field module adds the Email content fields for node content types.

Content fieldFormat
Label of Email fieldE-mail address

Supported settings

SettingComment
Default value
RequiredIf a field is Required, you must map it to a file column.
Number of valuesYou can map multiple values to multiple file columns or use a separator.

Number (CCK Core)

The Number module is fully supported.

Content fields

The Number module adds the Decimal, Float and Integer content fields for node content types.

Content fieldFormat
Label of DecimalDecimal number
Label of FloatFloat number
Label of IntegerInteger number

Supported settings

SettingComment
Default value
RequiredIf a field is Required, you must map it to a file column.
Number of valuesYou can map multiple values to multiple file columns or use a separator.
Allowed valuesYou can specify the value by key or label (see Allowed values).

Content fields

Text (CCK Core)

The Text module is fully supported.

Content fields

The Text module adds the Text content fields for node content types.

Content fieldFormat
Label of Text areaAny string
Label of Text fieldAny string
Input formatInput format

Supported settings

SettingComment
Default value
RequiredIf a field is Required, you must map it to a file column.
Number of valuesYou can map multiple values to multiple file columns or use a separator.
Text processingIf a text field is Filtered text, you will be able to map the Input format (see Input format).
Allowed valuesYou can specify the value by key or label (see Allowed values).
